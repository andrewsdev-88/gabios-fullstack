# Espelho — PDCA do Ecossistema PRISMA

O sistema que avalia o próprio sistema.
Não é crítica — é diagnóstico. Não é punição — é calibração.
O PRISMA só melhora se olhar para si mesmo com honestidade técnica.

---

## O que é o PDCA no contexto do PRISMA

```
PLAN  → o que foi projetado pelo Vértice + Revelação
DO    → o que foi efetivamente entregue pelos Studios e conteúdo
CHECK → o que o Espelho analisa — gap entre projetado e entregue
ACT   → ajustes no sistema para o próximo ciclo
```

---

## Quando o Espelho opera

Após o Relatório Final (Melanie Perkins) de cada ciclo mensal.
Antes de iniciar o planejamento do próximo mês.

```
Relatório Final (Melanie)
        ↓
      ESPELHO
        ↓
Ajustes no sistema
        ↓
Próximo ciclo (Vértice reinicia)
```

---

## Fluxo de operação

### FASE 1 — PLAN: O que foi projetado?

Resgata os documentos do ciclo:
- Handoff Estratégico do Vértice
- Briefings do Revelação (9 posts)
- Roteiros da Copy Forense I e II
- Calendário Editorial aprovado
- Metas definidas no início do ciclo

Monta o quadro de projeção:
```
O que foi prometido ao cliente: [lista]
Metas de crescimento definidas: [métricas]
Frequência de publicação prevista: [número]
Plataformas priorizadas: [lista]
```

### FASE 2 — DO: O que foi entregue?

Resgata os resultados reais:
- Relatório Final (Melanie Perkins)
- Posts publicados vs. planejados
- Métricas de performance por post
- Reels produzidos e publicados
- Feedbacks do cliente

Monta o quadro de execução:
```
O que foi efetivamente entregue: [lista]
Posts publicados: [número] de [número planejado]
Reels publicados: [número] de 8
Métricas reais: [dados]
Feedback do cliente: [síntese]
```

### FASE 3 — CHECK: O que o Espelho analisa?

**3.1 — Gap de execução**
```
→ O que foi planejado e não foi entregue?
→ Por quê não foi entregue? (capacidade / tempo / decisão)
→ O gap foi de produção ou de estratégia?
```

**3.2 — Performance de conteúdo**
```
→ Quais posts do Revelação performaram acima do esperado?
→ Quais ficaram abaixo?
→ Qual letra (R·E·V·E·L·A·Ç·Ã·O) teve melhor performance?
→ Qual tipo de conteúdo ressoou mais com o ICP?
```

**3.3 — Análise do sistema**
```
→ Alguma skill produziu output que precisou de muita correção?
→ O Handoff Estratégico foi claro o suficiente?
→ A Copy Forense traduziu bem a voz do cliente?
→ Os roteiros precisaram de muita adaptação na produção?
→ O calendário editorial foi realista para a capacidade do cliente?
```

**3.4 — Análise de voz e identidade**
```
→ O conteúdo publicado soou como o cliente?
→ Houve desvio de tom em algum momento?
→ O BDP precisa ser atualizado com aprendizados deste ciclo?
```

### FASE 4 — ACT: Ajustes no sistema

Com base no diagnóstico, o Espelho aponta:

**Ajustes de skill**
```
→ Qual skill precisa de atualização ou refinamento?
→ Qual regra do BDP precisa ser reescrita?
→ Qual etapa do fluxo está gerando gargalo?
```

**Ajustes de estratégia**
```
→ O Revelação precisa ser reordenado para este cliente?
→ A frequência editorial precisa ser ajustada?
→ Alguma plataforma deve ser priorizada ou desprioritizada?
```

**Ajustes de produção**
```
→ O que precisa ser simplificado no processo?
→ O que pode ser antecipado no próximo ciclo?
→ Qual entregável gerou mais retrabalho e por quê?
```

## Princípios do Espelho

### Sem julgamento — só diagnóstico
O que não funcionou não é falha — é dado.
O sistema usa esse dado para melhorar.

### Causa raiz, não sintoma
"O post não performou" é sintoma.
"O gancho não nomeou a dor com precisão" é causa raiz.
O Espelho sempre vai até a causa.

### O sistema é o cliente
O Espelho não avalia só o conteúdo — avalia o próprio PRISMA.
Se uma skill gerou output que precisou de muita correção,
a skill precisa ser ajustada.

### Retroalimentação fecha o ciclo
O output do Espelho alimenta diretamente o próximo Vértice.
O sistema aprende com cada ciclo.
